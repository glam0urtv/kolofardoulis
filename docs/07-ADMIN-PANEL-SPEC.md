# 07 — Admin Panel Spec

## Πρόσβαση

- Route: `/admin`, προστατευμένο με Supabase Auth, ρόλος `ADMIN`.
- Ξεχωριστή σύνδεση από τους πελάτες· συνιστάται 2FA για τον λογαριασμό
  ιδιοκτήτη.

## Dashboard

- Σύνοψη πωλήσεων (σήμερα/εβδομάδα/μήνα).
- Προειδοποίηση χαμηλού αποθέματος (κάτω από όριο ανά προϊόν, ρυθμιζόμενο).
- Τελευταίες παραγγελίες.

## Κατηγορίες

- CRUD, δενδρική δομή (parent/children) απεριόριστου βάθους.
- Drag-to-reorder.
- Εικόνα/εικονίδιο ανά κατηγορία.

## Προϊόντα

- CRUD πλήρες: όνομα, τιμή, περιγραφή (rich text), τύπος (Single/Booster
  Box/Booster Pack/Promo), stock, κατηγορία.
- Πολλαπλό upload εικόνων (drag & drop → Supabase Storage), αναδιάταξη.
- Ελεύθερα προσαρμοσμένα attributes (set, rarity, γλώσσα, κατάσταση) μέσω
  JSON πεδίου — ώστε νέα χαρακτηριστικά να μην απαιτούν migration.
- Ενεργό/Ανενεργό toggle (κρύβεται από το site χωρίς διαγραφή).

## Sections αρχικής σελίδας

- Λίστα `HomeSection` (Hero, Featured Categories, Banner, New Arrivals...).
- Προσθήκη/απόκρυψη/αναδιάταξη, με μικρή φόρμα ρυθμίσεων ανά τύπο section
  (π.χ. Hero: τίτλος, εικόνα/βίντεο φόντου, κουμπί CTA).
- Στόχος: ο ιδιοκτήτης αλλάζει την "διάθεση" της αρχικής χωρίς να αγγίζει
  κώδικα — αυτό είναι ελαφρύ page-builder, όχι πλήρες drag-and-drop εργαλείο
  (μπορεί να επεκταθεί σε επόμενη φάση αν χρειαστεί).

## Παραγγελίες

- Λίστα, αναζήτηση/φιλτράρισμα κατά κατάσταση.
- Λεπτομέρειες παραγγελίας, χειροκίνητη ακύρωση/επιστροφή χρημάτων (μέσω
  Stripe API), που ενεργοποιεί επιστροφή stock + audit log entry.

## Απόθεμα

- Χειροκίνητη διόρθωση stock (π.χ. φυσική απογραφή) με υποχρεωτικό λόγο.
- Κάθε αλλαγή γράφεται στο `InventoryAuditLog` — ποιος, πότε, πόσο, γιατί.

## Ρόλοι (Φάση 2, προαιρετικό)

- `ADMIN` (πλήρης πρόσβαση) vs. προαιρετικός περιορισμένος ρόλος `STAFF`
  (π.χ. μόνο διαχείριση παραγγελιών/αποθέματος, όχι τιμές/sections).
